Q: What is the idea?
A: I want to develop the vllm-cpu sub-project to add CPU Inference capabilities for the vLLM project. These capabilities can then be added to the Red Hat Inference Server product, providing our customers with the ability to do generative inference with either GPUs or CPUs. 2. Xeon 5 and 6 support the new ACE extensions for matrix multiplication in the x86 ISA. vllm-cpu adds a new run to the vLLM Inference Server to enable the Kernels for the Model Architectures supported to now run on CPU as well. There is no way to run vLLM on CPU today. So this feature doesn't add additional CPU capabilities, it adds the foundation for them.

Q: What is the project name for your idea?
A: vllm-cpu

Q: Why do you think this is relevant to Red Hat's current or future addressable market?
A: The new Blackwell and Vera Rubin GPUs exceed the power and capabilities of many data centers in many regions. In addition, existing data centers built for traditional workloads are full of Intel Xeon 5 and Xeon 6 servers that aren't fully utilized that and combined with their existing Kubernetes deployments, this provides a compelling alternative for certain types of workloads. Accelerator optionality is a key aspect of Red Hat's hybrid platform value proposition.
